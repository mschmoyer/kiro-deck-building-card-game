# User Context - Kiro Deck Building Game

## Technology Preferences
- **Framework**: Next.js
- **Language**: JavaScript (avoiding TypeScript for speed)
- **UI Library**: Elegant UI library (shadcn/ui or similar)
- **Timeline**: 2-hour workshop build

## Game Mechanics Decisions
- **Win Condition**: 10 points, earned by solving bugs/features over 20 turns
- **Card Mechanics**: 
  - Cards have costs to create purchase friction
  - Cards provide "Task" points
  - Some cards may provide "Specs" 
  - Card draw is valuable
- **Bugs & Features**: 
  - Separate deck from player cards
  - Always 3 active bugs/features on the board
  - Each requires # of Tasks to complete
  - May require Specs as well
- **No Physics**: No gravity or movement mechanics needed
